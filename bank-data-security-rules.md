# 银行业数据安全规范

依据《个人金融信息保护技术规范》（JR/T 0171—2020）及现行监管要求，银行业隐私数据按敏感程度从高到低分为 **C3、C2、C1** 三个级别。

> **核心原则**：在未经加密和通道保护的情况下，所有级别的个人金融信息均**默认不得直接传输**。
> - **C3**：最高敏感级别，严禁明文传输，必须加密，未登录状态不得展示，登录状态不得明文展示
> - **C2**：通过公共网络传输时须使用加密通道或数据加密
> - **C1**：无强制加密要求，但建议内网传输并遵循最小权限原则

根据《中国人民银行业务领域数据安全管理办法》（2025年6月30日起施行），不得通过邮件、即时通讯等非专用渠道传输高敏感性数据；公开高敏感性数据项原则上须作脱敏处理。

## 监管依据说明

根据《中国人民银行业务领域数据安全管理办法》（2025年6月30日起施行），银行应使用专用渠道传输高敏感性数据，不得通过邮件、即时通讯等非专用渠道传输。该办法还明确规定，数据处理者不得公开用于身份鉴别的数据项，公开其他高敏感性数据项原则上须作脱敏处理。

同时，根据JR/T 0171—2020《个人金融信息保护技术规范》，C2、C3类别信息通过公共网络传输时，均应使用加密通道或数据加密的方式进行传输，保障个人金融信息传输过程的安全。C3类别中的支付敏感信息，其安全传输技术控制措施应符合有关行业技术标准与行业主管部门有关规定要求。此外，处于未登录状态时，不应展示与个人金融信息主体相关的C3类别信息；处于已登录状态时，除银行卡有效期外的C3类别信息不应明文展示。

## 附加说明

1. **敏感级别**以《个人金融信息保护技术规范》（JR/T 0171—2020）为依据。
2. **是否可以直接传输**一栏中的规则依据包括《中国人民银行业务领域数据安全管理办法》（2025）和JR/T 0171—2020。
3. 机构与企业数据中的个人相关信息（如法人代表信息、授权经办人信息）应归入对应个人金融信息类别，按相应级别保护。
4. 同一信息在不同服务场景中可能处于不同的类别，需依据具体场景对信息的类别进行识别并采取相应保护措施。
